1. Web & DOM Runtime Adapter

Grain isn't just a parser; it is a live rendering engine. The @grain.sh/web package includes the WebAdapter, which observes Grain strings and reactively updates the DOM tree. This is highly valuable because it converts raw LLM text streams directly into safe, encapsulated UI components without React reconciliation overhead.

Why it's valuable

  • XSS Safety: Grain's lexical parser sanitizes input at the AST level. It is vastly safer than injecting Markdown innerHTML.
  • Partial Stream Rendering: It gracefully handles unclosed tags (e.g., <stream speed="fast">Hel...) by holding them in partial state, allowing smooth typewriter effects.

Implementation

import { WebAdapter } from '@grain.sh/web';

const adapter = new WebAdapter();
const container = document.getElementById('chat-window');

const incomingStream = '<think>Searching...</think><stream>Found it!</stream>';

await adapter.render(incomingStream, { container });

2. Agent-to-Agent Swarms

In multi-agent architectures (e.g., LangGraph, AutoGen), agents often struggle to pass strict state using arbitrary JSON envelopes. Grain solves this by acting as the universal envelope.

Why it's valuable

A "Chief" agent can emit a <context> block containing the exact state, and delegate to a "Sub-Agent" via the <agent> primitive. The sub-agent inherently parses the context without needing a custom proprietary schema definition.

Implementation

const chiefOutput = `
  <agent name="researcher" priority="high">
    <context>
      User wants to know the latest news on Quantum Computing.
    </context>
    <instruction>Execute web search and return a summary.</instruction>
  </agent>
`;

import { parseAgentTags } from '@grain.sh/agent';
const delegations = parseAgentTags(chiefOutput);
delegations.forEach(worker => orchestrator.spawn(worker.name, worker.context));

3. Model Context Protocol (MCP) Runtime

MCP standardizes tool definitions, but how does the LLM actually call those tools in a unified way? Instead of using proprietary OpenAI tool formats or Anthropic XML formats, models can be instructed to output standard Grain <tool> tags.

Why it's valuable

You can hot-swap the underlying LLM (e.g., moving from GPT-4 to Claude 3.5) without changing your application's tool execution harness. The @grain.sh/mcp adapter standardizes the extraction.

Implementation

import { MCPAdapter } from '@grain.sh/mcp';
import { executeLocalTool } from './my-tools';

const parser = new MCPAdapter();
const llmResponse = '<tool name="read_file" args=\'{"path":"/tmp/log"}\'>';

const pendingTools = parser.extractTools(llmResponse);

for (const tool of pendingTools) {
    const rawResult = await executeLocalTool(tool.name, tool.args);
    
    // Feed the result back to the LLM via Grain standard
    const reply = `<tool_result name="${tool.name}" status="success">${rawResult}</tool_result>`;
    llm.send(reply);
}

4. Iterative Chunk Streaming

Grain's core parser is an iterative state machine. This means you do not need to buffer the entire LLM response before parsing it.

Implementation

import { GrainParser } from '@grain.sh/core';

const parser = new GrainParser();

// Assuming 'stream' is a standard async iterable from an LLM API
for await (const chunk of stream) {
    const parsedASTNodes = parser.append(chunk);
    
    // Immediately act on new nodes as they arrive
    for (const node of parsedASTNodes) {
        if (node.type === 'think') {
            console.log('Agent is reasoning: ', node.content);
        }
    }
}
